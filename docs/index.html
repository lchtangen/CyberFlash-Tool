// what it does

Everything in One Place

CyberFlash eliminates the Android modding fragmentation problem. No more switching between five terminals, three GUIs, and a stack of outdated scripts. Every feature you need is built-in, tested, and ready to use.

⚡

Flash Engine

Full fastboot and ADB sideload ROM flashing with dry-run mode, step tracker, danger confirmation, and real-time ANSI log output. A/B slot management and vbmeta patching included.

🔓

Root Management

Detect and install Magisk, KernelSU, and APatch. Boot image patch workflow, module browser, DenyList manager, dm-verity/force-encryption control, and banking-safe preset.

📦

ROM Library

Curated ROM source browser with AI-powered trust scoring across availability, safety, speed, and reputation. Resumable chunked downloads with SHA-256 verification and history persistence.

🔬

Device Diagnostics

31 ADB diagnostic commands across 6 categories. Real-time device health card (battery, temperature, storage, RAM). Streaming logcat with priority filter (V/D/I/W/E/F/S).

🏢

Multi-Brand Flash

Native support for Samsung (Heimdall/PIT), Xiaomi (MIUI/HyperOS + ARB check), Google Pixel (factory images + OTA), and Motorola (RSA rescue + flashfile.xml). EDL emergency download mode.

💾

Backup & Restore

ADB backup mode and media pull mode with abort support. Full app backup orchestration with progress reporting and background worker threads to keep the UI responsive.

🔑

Token Vault

AES-256-GCM encrypted credential storage with OS keychain integration (keyring) and PBKDF2 key derivation. Securely store OEM unlock tokens, API keys, and device passwords.

🛡️

Play Integrity Checker

3-tier Play Integrity / SafetyNet verification (BASIC / DEVICE / STRONG) with history log in JSON. Understand exactly where your device's attestation stands after any modification.

⚙️

Kernel Manager

AnyKernel3 flash via sideload or ADB push. Boot image backup and rollback. Kernel version verification after flash. Never get stuck on a bad kernel again.

🎨

Boot Animation

Parse, preview, install, backup, and reset bootanimation.zip. Frame extraction for inspection. Personalize your device's boot sequence without any extra tools.

📡

Wireless ADB

Pair and connect via Wi-Fi (Android 11+ TCP pairing). No USB required for day-to-day device management once paired. Connection state persisted across sessions.

🖥️

Payload Extractor

Inline protobuf-based OTA payload.bin extractor with zero external dependencies. Extract individual partition images from any Android OTA update package directly in the app.

// intelligence layer

AI-Powered Features

CyberFlash integrates AI assistance at every layer — from intelligently scoring ROM download sources to analyzing device errors in plain English. The AI features work alongside the tool; they never replace your control.

ROM Source Trust Scoring

AI evaluates ROM download sources across four dimensions: Availability, Safety, Download Speed, and Community Reputation. Flags suspicious or dead sources automatically.

Error Analyzer

When a flash or ADB command fails, the AI error analyzer parses the error output and provides a plain-English explanation with suggested next steps — no more Googling cryptic fastboot errors.

AI Assistant Panel

Built-in AI chat assistant wired into the main window. Ask questions about your device, current flash procedure, root status, or any Android modding topic without leaving the app.

Device Health Scoring

AI-driven health scoring combines battery wear, thermal data, storage health, and RAM pressure into a single actionable score with per-metric breakdowns and recommendations.

Gemini Integration

Optional Google Gemini client for enhanced natural language queries. Securely stored via the encrypted Token Vault — your API key never touches plaintext storage.

Security Auditor

AI-assisted security audit of your device configuration: SELinux state, dangerous permissions, developer options exposure, USB debugging state, and network security posture.

// how it's built

Clean Architecture

CyberFlash enforces strict layer boundaries so every module is independently testable. UI never calls ADB directly. Workers never block the main thread. Core logic has zero Qt imports.

ui/ Pages, dialogs, panels, widgets, themes — Qt only. No subprocess or I/O.
workers/ QObject + moveToThread. Async bridge between UI and core. Signals/slots only.
services/ Own workers, expose high-level signals to UI. DeviceService, RomLinkService, AIService.
core/ Pure Python. No Qt widget imports. Flash engine, ADB, root, brand managers. Testable in isolation.
models/ Dataclasses only. DeviceProfile, FlashTask, EdlTask, RomSource.
utils/ stdlib + platform helpers. No business logic.

Quick Install

CyberFlash runs on Linux, macOS, and Windows. Python 3.12+ and pip are the only prerequisites.

# 1 — Clone the repository
git clone https://github.com/lchtangen/CyberFlash-Tool.git
cd CyberFlash-Tool

# 2 — Create a virtual environment and install
python -m venv .venv
source .venv/bin/activate # Windows: .venv\Scripts\activate
pip install -e ".[dev]"

# 3 — Launch
python -m cyberflash
# Run the test suite
pytest tests/

# Lint check
ruff check src/

Linux users: install udev rules for EDL mode support → sudo cp resources/udev/51-cyberflash-edl.rules /etc/udev/rules.d/

// open source

Built for the Community

CyberFlash is MIT-licensed and designed from day one for community contribution. Device profiles are plain JSON files. Core logic is pure Python with no compile cycle. The test suite makes it safe to contribute without breaking things.

Add Device Profiles

Device support is defined in JSON files under resources/profiles/. Add your device by creating a JSON profile — no Python required. Schema is fully documented.

Fix Bugs & Features

Fork, branch, test, and PR. The strict layer architecture means changes are isolated. Run pytest tests/ and ruff check src/ before submitting.

Write Tests

Core modules are pure Python — easy to test without a device. We're targeting 900+ tests. Each new module needs coverage before merge.

Report Issues

Encountered a device-specific edge case, a bad flash sequence, or a UI bug? Open an issue on GitHub with your device codename and the log output.

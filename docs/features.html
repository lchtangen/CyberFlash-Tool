Features

Every feature in CyberFlash — organized by tier and module. Production-ready features are fully tested. Planned features are clearly marked.

// tier 1 — production ready

Core Flash Engine

The heart of CyberFlash. Every operation here is battle-tested with dry-run validation before any destructive action executes.

ROM Flash (fastboot)

Sequential fastboot partition flash with step tracker. Supports boot, system, vendor, vbmeta, dtbo, and custom partition targets.

  • Dry-run mode — simulate before commit
  • Danger confirmation dialogs
  • ANSI-colorized real-time log panel
  • Flash journal with rollback support

ROM Flash (sideload)

ADB sideload for TWRP and stock recovery. Progress tracking and error detection throughout the sideload sequence.

  • TWRP sideload support
  • Stock recovery sideload
  • Progress bar + ETA display

A/B Slot Management

Full A/B partition scheme support. Switch active slot, inspect current slot, and ensure flashed partitions land on the correct slot.

  • Active slot display and switch
  • Slot-aware flash sequencing
  • Post-flash slot verification

Wipe Operations

Granular wipe control with confirmation for each dangerous operation. Never wipe the wrong partition by accident.

  • Cache / Dalvik / Data / System
  • Vendor / Userdata wipe
  • Per-wipe danger confirmation

Device Detection

Real-time ADB and fastboot hot-plug polling via DeviceService. Multi-device selector in the title bar. Device info page with full hardware summary.

  • Hot-plug detect (ADB + fastboot)
  • Multi-device selector UI
  • Model, Android, BL status, serial

Preflight Checks

Automated pre-flash safety verification: bootloader state, battery level, storage space, ADB authorization, and profile compatibility.

  • Battery level gate (≥30%)
  • Storage space check
  • ADB authorization verify
// tier 2 — production ready

Root & Security

Complete root lifecycle management. From initial installation through module management to Play Integrity compliance.

Root Detection

Automatically detects root method on every device connect. Distinguishes between Magisk, KernelSU, APatch, other root, and unrooted states.

Magisk Boot Patch

Full automated boot patching: push original boot image → launch Magisk → poll for patched image → pull → flash via fastboot. All steps logged.

Module Manager

Browse, install, enable, disable, and remove Magisk and KernelSU modules. Online repo browser with search and filter. Install from local ZIP.

DenyList & Root Hide

Per-app DenyList configuration via Magisk. Banking-safe preset applies the correct hiding configuration for major financial app packages automatically.

AnyKernel3 Flash

Flash custom kernels via AnyKernel3 using sideload or ADB push. Automatic boot image backup before every flash. One-click rollback to saved boot image.

Play Integrity Checker

3-tier verification: BASIC (device passes basic checks), DEVICE (hardware attestation), STRONG (full hardware-backed key attestation). JSON history log.

Token Vault

AES-256-GCM encrypted storage for OEM unlock tokens, API keys, and device credentials. Backed by OS keychain (keyring) with PBKDF2 master key derivation.

dm-verity & Encryption

Toggle Android Verified Boot (dm-verity) and force-encryption from the root page. Essential for certain custom ROM workflows that require these to be disabled.

// tier 3 — production ready

ROM Library & Downloads

Find, evaluate, download, and verify ROMs without leaving the app. AI-powered trust scoring ensures you only download from reliable sources.

AI Trust Scoring

Every ROM source is scored across four AI-evaluated dimensions: Availability (uptime/reliability), Safety (malware/tampering risk), Speed (server throughput), Reputation (community signals).

Resumable Downloads

Chunked HTTP/HTTPS download with Range header support. Interrupted downloads resume from where they stopped — no re-downloading gigabytes from scratch.

SHA-256 Verification

Every downloaded ROM is automatically verified against its SHA-256 checksum post-download. Failed verification triggers a re-download prompt, not a silent failure.

payload.bin Extractor

Extract individual partition images from OTA payload.bin files directly in the app. Inline protobuf parser — no external tools or Python packages required.

Download History

Per-device download history persisted to a JSON flat file. Already-downloaded ROMs are detected on page load. History survives app restarts and system reboots.

Source Flagging

Low-trust sources are flagged with visible warnings. Flagged sources require explicit user acknowledgement before download begins — no accidental downloads from sketchy mirrors.

How to Contribute

CyberFlash is MIT-licensed. The architecture is designed so any developer — Python expert or beginner — can contribute meaningfully.

01

Add a Device Profile

Create a JSON file in resources/profiles/your-brand/codename.json. Follow the schema at resources/profiles/schema.json. No Python required.

02

Fix a Bug

Fork the repo, create a branch, reproduce the bug in a test, fix it, verify pytest tests/ passes, open a PR with the device codename and log output.

03

Write Tests

All core/ modules are pure Python — testable without a physical device. We target 900+ tests. Each new feature needs coverage before merge.

04

Review Standards

Run ruff check src/ and mypy src/ before opening a PR. Type hints are required on all public functions. Line length: 100 chars.
